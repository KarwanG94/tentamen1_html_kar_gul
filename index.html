Responsiv Webbdesing

Responsive webbdesign innebär att webbsidan anpassar sig efter skärmens storlek, Att utvecklaren designar webbsidan så att det passar till vanliga Monitors ända ner till Smartphones. Att webbsidans innehåll anpassar sig beorende på vilken sorts skärm man använder.

1. Man kan använda sig av Viewports och % för att anpassa innehållet på sidan efter skärmens storlek.

2. Man kan också använda sig av Brytpunkter (Media Queries)

Brytpunkter (Media Queries) är ett sätt för utvecklaren att anpassa webbsidan beroende på vilken storlek skärmen är på. Om du vill att webbsidan ska ändras på något sätt efter att skärmen gott ner till 700px så använder man sig av Brytpunkter.(Du kan använda flera brytpunkter än bara 1)

Bootstrap

Bootstrap är ett hjälpmedel för utvecklarna. Det används för att kalla på t.ex. olika "klasser" som är redan färdiga. detta gör kodandet smidigare och enklare. t.ex. Om du vill göra en navigation på toppen av sidan som beter sig på ett visst sätt/ser ut på ett visst sätt så kan du använda dig av bootstrap för att kalla på en klass som redan har den koden färdig.

Man kan implementera bootstrap Lokalt Vilket innebär att man laddar ner en specifik fil med dem klasser man vill åt på sin dator.

Man kan också implementera boostrap via "CDN" där man länkar sitt projekt med boostrap online och inte lokalt.

Kod Granskning

  • "style" taggen bör tas bort från html filen och vara på en separat .CSS fil pga. det blir lättare och mer organiserat i längden. (+ att du redan har en länk till en extern css fil).
  • "Tabba" in innehållet under Body Taggen för att det ska bli mer organiserat och hålla en bättre struktur. (Detta gäller även själva innehållet på t.ex. section taggen också.)
  • Du kan använda dig av korrekt html taggar på t.ex. Navigationen istället för en div.
  • Du har en h3 istället för h1 som första rubrik.
  • Du använder dig av "div" taggar istället för paragrafer (p) när du presenterar sidan. (och i din footer). När du skriver brödtext så använder du dig av (p) taggen. själva P taggen kan därefter ligga i div taggen (som du gjort i footern)
  • Glöm inte bort att lägga till "alt" i din img tag, ifall bilden inte kan laddas.
  • Igen, du lägger till en "style" i din div tag istället för att använda dig av din externa css fil
  • Glöm inte stänga taggarna (som på Body) med "/"
